## Why A2UI is more efficient

OpenUI Lang is a compact DSL — but the LLM still has to describe the **structure** of the UI (layout, component nesting, argument order). A2UI uses a **compiled vocabulary**: the LLM emits a component name + a handful of semantic fields; all layout, styling, and rendering logic is precompiled into the renderer. There are zero tokens spent on structure.

```
OpenUI Lang (pricing-page, 1,217 tokens):
  root = Stack([hero, pricingRow, tableTitle, featureTable, faq], "column", "xl")
  pricingRow = Stack([basicCard, proCard, enterpriseCard], "row", "l", "stretch", "start", true)
  basicCard = Card([basicHeader, basicPrice, ...], "card")
  ...

A2UI (pricing-page, 183 tokens):
  {"component":"pricing_tier_group","tiers":[...]}
  {"component":"feature_matrix","features":[...],"tiers":[...]}
  {"component":"faq_accordion","items":[...]}
```

## Caveats

- v1 used approximate mappings for `contact-form` (callout + checklist) and `settings-panel` (status_dashboard). v2 uses native `form` and `form_switch_group` atoms sourced from OpenUI — those gaps are now closed. See `vendors/openui/MANIFEST.md`.
- **All four comparison formats** (OpenUI Lang `.oui`, YAML, Vercel JSON-Render `.vercel.jsonl`, Thesys C1 JSON `.c1.json`) are published by Thesys in the same repository (`github.com/thesysdev/openui/benchmarks/samples`) under MIT. "Vercel JSON-Render" and "C1 JSON" are Thesys's format labels for their own benchmark baselines — they are not formats published by Vercel or any other vendor. A single attribution to Thesys covers all four formats.
- OpenUI Lang figures are the **exact reference samples** from the OpenUI benchmark repo — not regenerated by an LLM, so both sides are measured at their authored minimum.
- Token counts use `cl100k_base` (GPT-4 encoder). OpenUI's published benchmarks use `tiktoken` with the `gpt-5` encoder; results may differ slightly.
